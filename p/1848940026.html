4. Supplementary Group IDs

UNIX系统的设计之初并没有包含supplementary group. 单个用户只属于一个primary group, 因此我们需要调用newgrp()来切换group. 当检查文件访问权限时, kernel也只需对比user的primary group ID和文件所属的group ID是否相等. 当执行完毕操作后, user还需再次调用newgrp()切换回原本的group.
但user和group之间的关系不应为多对一, 而是多对多, 每个group可包含多个user, user也可属于多个group, 因此从4.2BSD开始添加supplementary group ID的概念. user仍然拥有其primary group ID, 但也同时拥有多个supplementary group ID. 当检查文件访问权限时, 不仅要对比user的primary group ID, 还需要对比所有的supplementary group ID. 因此不再需要显式地修改primary group ID.

5. Other Data Files

上述只提到了两类文件: password file和group file. 其他UNIX系统的文件还包括: network service(/etc/services), internet protocols(/etc/protocols). UNIX提供了三类函数来访问这些文件:

  1. get function: read the next entry of file.
  2. set function: rewind the file
  3. end function: close the file

6. Login Accounting

UNIX系统提供了两个文件:

  1. utmp file: 记录当前所有登录的用户
  2. wtmp file: 记录所有登录和登出信息

用户登录时, login()会创建一个utmp structure并写入utmp file和wtmp file; 当用户登出时, logout()会删除utmp file中的utmp structure, 并新建一个utmp structure写入wtmp file. 由于utmp structure在不断添加新的field, utmp和wtmp文件也变得越来越大.
