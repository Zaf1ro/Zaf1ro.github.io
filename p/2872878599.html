2. 子类重载的权限问题

  1. Java子类重写方法时, 不能降低方法的访问权限, 子类继承父类的访问修饰符要比父类的更大. 假如父类的方法为protected, 那么子类只能是protected或者public
  2. 如果父类的函数为private, 那么就对子类不可见, 这时并不构成重载条件
  3. 子类能继承父类的所有成员, 可通过反射得到所有成员

3. 构造顺序

类加载过程中:
父类静态代码块 -> 子类静态代码块 -> 父类构造代码块 -> 父类构造函数 -> 子类构造代码块 -> 子类构造函数

4. 枚举的构造函数

枚举的构造函数默认为private, 说明不允许显式调用

5. HashTable和HashMap的区别

  1. 继承不同
    HashTable继承自Dictionary
    HashMap继承自AbstructMap
  2. 是否并发
    HashTable支持并发, HashMap不支持并发
  3. key和value
    HashTable的key和value都不允许为null
    HashMap中null可作为key, 但只能存在一个. 可有一个或多个key对应null value.
  4. 遍历方式
    HashTable和HashMap都使用了Iterator, 但HashTable还是用Enumeration的方式遍历
  5. hashCode
    HashTable使用对象的hashCode, 而HashMap重新计算hash值
  6. 默认大小和扩容方式
    HashTable默认大小为11, 扩容方式为2*old+1
    HashMap默认为16, 扩容方式为2的指数大小

9. 抽象类

抽象类并不是只能有抽象方法, 也可以有成员变量和普通的类方法. 但有三点和普通类不同:

  1. 抽象方法必须是public或protected
  2. JDK 1.8之前默认为public
  3. JDK 1.8时默认为default
  4. 抽象类不能创建对象
  5. 子类必须实现父类的抽象方法, 否则只能声明为抽象类
  6. 抽象类中也可以没有抽象方法

14. interface方法修饰

  1. JDK 1.8之前, 接口方法只能被public修饰
  2. JDK 1.8时, 接口方法可被public和default修饰
  3. JDK 1.9时, 接口方法可被public, default, private

内部接口的方法只能为public, default, protected, private, abstract, static

15. 抽象类和接口的异同

不同之处:

  1. 抽象类可有构造方法, 接口不能
  2. 抽象类可包含非抽象的普通方法, 接口不能
  3. 抽象类可有普通成员变量, 接口不能
  4. 抽象类中抽象方法的访问类型可为public, protected和默认类型, 接口的方法只能为public
  5. 抽象类可有静态方法, 接口不能
  6. 一个类可实现多个接口, 只能继承一个抽象类
    相同之处:
  7. 抽象类和接口都可包含静态成员变量
  8. Java8中的接口可有default方法

22. 内部类和外部类的访问机制

  1. 外部类不能用static来修饰, 内部类可用static修饰
  2. 外部类只能用public和默认访问权限来修饰, 内部类可用public, private, protected和默认访问权限
  3. 局部内部类
  4. 不能从方法外部调用
  5. 不能访问非final局部变量(jdk1.8可以访问非final局部变量)
  6. 不能设置任何访问权限(public, protected, private和static都不可以), 但可以设置为final
  7. non-static内部类
    1. 内部类可设置访问权限(public, protected,private都可以)
    2. 内部类可访问外部类的变量和方法
    3. 内部类在外部类中留下一个引用, 外部类可引用内部类
    4. 创建内部类对象需要外部类对象
  8. static内部类
  9. 内部类可设置访问权限
  10. 不能使用外部类的non-static成员变量和方法
  11. 创建不需要依赖外部类
  12. 外部类可创建内部类实例并访问变量和方法
  13. 匿名内部类
    1. 可访问外部类的变量和方法
    2. 没有构造方法
    3. 没有访问修饰符
