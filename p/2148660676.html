2. 字符串拼接

  1. 拼接静态字符串时, 尽量使用”+”, 因为编译器会自动优化为长字符串

  2. 拼接动态字符串时, 可使用StrinBuilder.append(). 不要用”+”, 因为连接两个String都要创建一个临时String来存储结果.

4. 静态域

静态域内的变量或语句与类实例无关, 并会在加载类时对静态域进行初始化, 所有静态初始化语句和静态初始化块都被按照类定义顺序执行, 默认初始值为0, false或null. 静态域与classloader绑定, 如果两个程序使用不同的classloader, 则他们拥有两个互相独立的静态域.

5. toString()

  1. 调用toString()可获得对象的字符串

  2. toString()会在某些情况下隐式调用 (对象与字符串用+连接, 或直接在System.out.print()函数中出现)

  3. 对象可重载Object的toString(), 例如StringBuilder.toString()

  4. 数组不支持toString(), 必须调用静态函数Arrays.toString()

7. Type Erasure(类型擦除)

代码中定义List和List编译后都会成为List, JVM只会看到List, 并不会看到类型信息, 因为所有元素都会被擦除为Object

所以元素类型的定义只是为了向Collections中加入元素时进行类型检查. 若定义时不写元素类型, 则编译器不会进行类型检查

12. 取整

Java有三种取整方式:

  • Math.floor(double d): 小于等于d的最大整数
  • Math.ceil(double d): 大于等于d的最小整数
  • Math.round(double d): 最接近d的整数

round()需要注意正负值:

  • d > 0:
    • 小数点后一位的值为[0, 4], d取整
    • 小数点后一位的值为[5, 9], d取整后+1
  • d < 0:
    • 小数点后一位的值为[0, 5], d取整后+1
    • 小数点后一位的值为[6, 9], d取整

13. 并发编程的同步器

  1. Semaphore: 完成信号量控制, 可控制某个资源同事被访问的个数, 并通过acquire()获得一个访问许可. 如果没有则等待, release()释放许可
  2. CyclicBarrier: 只有await()方法, 每调用一次计数则减1, 并阻塞当前进程. 当计数器归0时阻塞解除, 所有阻塞的线程都开始运行.
  3. CountDown: 在CountDown.await()归0之前阻塞当前线程

14. monitor

  • Java使用monitor来保证同一时间只有一个线程可以访问一段代码或数据
  • 同时只能有一个线程获得某个对象的monitor
  • 只有拥有了该对象的monitor才能调用对象的notify()和notifyAll(), 否则会抛出java.lang.IllegalMonitorStateException
  • 一个线程想拥有一个对象的monitor的方法有三种:
    • public synchronize a(){}
    • synchronize(obj){}
    • public static synchronize b(){}

15. try-catch

try可以对应多个catch, 但只会匹配第一个相符的catch. 而且会从上向下依次匹配, 所以应将异常范围小的放在前面.

16. finally

finally块内的语句会在try结束前被执行, 所以finally中不应return. try-catch-finally的执行流程如下:

  • try中出现的异常catch能捕获到, 转由catch处理
  • try中出现的异常catch无法捕获, 转到finally
  • catch中无论是否出现异常, 都会结束后转到finally

17. ThreadLocal

ThreadLocal的作用是为线程提供一个可共享的局部变量. “可共享”是因为该线程的所有函数都可以访问该变量; “局部变量”是因为其他线程无法访问和修改该变量. 所以ThreadLocal避免了全局变量的一个缺点: 所有线程都可访问. 并且减少了同一个线程内多个函数使用公共变量传递的不便.

18. 方法签名

当函数为同类同名时, 会出现重载问题. Java通过方法签名来确定唯一的方法. 由于重载只与参数类型, 参数个数和参数顺序有关, 所以签名必须包含这些信息

以下是方法签名中特殊字符/字母的含义

V void 一般用于表示方法的返回值
Z boolean
B byte
C char
S short
I int
J long
F float
D double
[ 数组 以[开头, 配合其他的特殊字符, 表示对应数据类型的数组, 几个[表示几维数组
L 引用类型 以L开头 ;结尾, 中间是引用类型的全类名

20. 隐式转换

从小到大, 可隐式转换, 数据类型自动提升:
byte, short, char -> int -> long -> float -> double
Java有四种情况下进行隐式转换:

  1. 算数运算
  2. 赋值表达式
  3. 函数调用中参数传递
  4. 返回表达式类型
    但从大到小只能强制转换

但final修饰的变量不会进行自动隐式转换

21. Integer和==

当赋予Integer一个int类型时, 会调用Integer.valueOf()来将int转换为Integer. 但valueOf()会将[-128, 127]之间的int数字放入IntegerCache中, 而范围之外的int数字会赋予一个新的内存空间.

22. volatile

Java内存模型规定, 所有变量都要存在主存(物理内存)当中. 每个线程都有自己的工作内存(类似于告诉告诉缓存). 线程对变量的所有操作都必须在工作内存中进行.

上述语句需要先将10写入工作内存中, 然后再写入主存中. 这样就会导致各个线程中的工作内存放置的数据不相同. volatile用来修饰被不同线程访问和修改的变量, 能保证共享变量能立即刷新到主存中. 当其他线程需要读取时, 会从主存中读取新值.
volatile也能禁止指令重排序优化. 因为编译器对指令顺序的重新排序可能导致程序错误.
Volatile 变量具有synchronized的可见性特性, 但是不具备原子特性
