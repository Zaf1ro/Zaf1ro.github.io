1. static

  • 全局变量
    作用域是整个源程序
  • 静态全局变量:
    静态全局变量限制了作用域, 变为单个源文件.
  • 局部变量
    生存周期为所在函数的生存周期
  • 静态局部变量
    生存周期变为整个进程的生存周期
  • 静态成员变量
    静态成员的生存周期取决于进程的生存期(与全局变量, 静态变量相同), 非静态成员的生存期取决于对象的生存期. 静态成员需要在类体外初始化. 静态数据成员可用类名调用, 也可被类对象调用

2. const与#define

使用const比使用define有一下几种好处:

  1. const会进行数据类型检查, 而define不会
  2. const效率高. 因为const定义的常量, 没有在内存中存储, 而是在符号表(symbol table)中. 每次访问这个数据的时候, 少了从内存中读取和存储过程, 所以效率高.
    尽量使用const常量

3. struct in C++

C++的struct与C的struct完全不同. C++中的struct已经十分接近class. 除了访问权限无法设置, 继承(inheritance)和多态(polymorphism)都具备了.

4. 面向对象五个原则

  1. 单一职责原则(Single-Resposibility Principle): 一个类, 最好只做一件事, 只有一个引起它的变化.单一职责原则可以看做是低耦合, 高内聚在面向对象原则上的引申, 将职责定义为引起变化的原因, 以提高内聚性来减少引起变化的原因.
  2. 开放封闭原则(Open-Closed principle): 软件实体应该是可扩展的, 而不可修改的. 对扩展开放, 对修改封闭的.
  3. Liskov替换原则(Liskov-Substituion Principle): 子类必须能够替换其基类.这一思想体现为对继承机制的约束规范, 只有子类能够替换基类时, 才能保证系统在运行期内识别子类, 这是保证继承复用的基础.
  4. 依赖倒置原则(Dependecy-Inversion Principle): 依赖于抽象. 具体而言就是高层模块不依赖于底层模块, 二者都同依赖于抽象; 抽象不依赖于具体, 具体依赖于抽象.
  5. 接口隔离原则(Interface-Segregation Principle): 使用多个小的专门的接口, 而不要使用一个大的总接口

6. 初始化列表

使用初始化列表可少调用一次default constructor, 所以可提高效率, 并减少错误. 以下三种情况必须使用初始化列表:

  1. 类成员所在的类没有默认构造函数
  2. const修饰的类成员或引用成员数据
  3. 子类初始化父类的私有成员

static全局变量不属于某个类对象, 所以不能在类内初始化

8. 类在堆上和栈上的区别

调用Object obj不能保证类对象分配到栈中, 因为C++遵循自动存储(automatic storage). C++会根据context决定存储位置.

  1. Object obj存储在栈中

  2. Object obj存储在堆中

  3. Object Obj在静态存储区

堆与栈上类对象的区别在于:

  1. 性能
    • 栈中的类对象的分配时间是常量
    • 堆上的类对象的分配时间不定, 因为需要操作系统追踪内存的可用区域
  2. 生存周期
    • 栈上的类对象生存周期与上下文有关
    • 堆上的类对象则有更长的生存周期

11. clone和fork

  • fork(): 复制时将父进程的所以资源都通过复制数据结构进行了复制, 然后传递给子进程. 所以不带参数.
  • clone() 函数则是将部分父进程的资源的数据结构进行复制，复制哪些资源是可选择的，这个可以通过参数设定，所以 clone() 函数带参数，没有复制的资源可以通过指针共享给子进程

27. 请求stack空间的四种方法

  1. void* malloc(unsigned int size)
    • 在stack中分配一段长度为size的连续空间
    • 成功时返回首地址, 失败时返回NULL
    • 不会初始化内存
  2. void* calloc(unsigned int num, unsigned int size)
    • 根据数据个数和数据类型所占字节数, 分配size*num大小的连续空间
    • 成功时返回首地址, 失败时返回NULL
    • 会自动初始化内存为0
  3. void* realloc(void *ptr, unsigned int size)
    • 分配一段长为size的内存空间, 并把内存空间的首地址赋值给ptr
    • 内存分配失败则返回NULL
    • 不会初始化内存
  4. void* operator new (std::size_t size)
    • 自动计算需要分配的内存空间大小, 并调用类的构造函数
    • 成功时返回首地址, 失败时抛出bad_alloc异常
    • 不对类中的内置类型变量进行初始化
